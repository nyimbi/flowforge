# Flowforge 0.1.0 — upgrade checklist

> First versioned release. Closes 77 audit-2026 findings under
> DELIBERATE-mode signoff. **Read this before upgrading.** Includes one
> SECURITY-BREAKING change (E-34) with a one-minor opt-in bridge.

## TL;DR

1. Set `FLOWFORGE_SIGNING_SECRET` to a real secret on every host.
2. Run `flowforge pre-upgrade-check signing` and confirm `PASS`.
3. Apply the audit-pg ordinal backfill on every existing PG-backed
   environment (`scripts/ops/audit-2026-soak.sh` documents the runbook;
   the migration itself is `python -m flowforge_audit_pg.migrations.audit_ordinal_backfill`).
4. Audit your CSRF wiring: `secure=False` now requires `dev_mode=True`
   or you'll get `flowforge_fastapi.ConfigError` at startup.
5. Audit any `register_op(...)` call site: the registry is frozen at
   module-init in 0.1.0; any post-startup register attempt now raises
   `RegistryFrozenError`.
6. Audit your hub `install(...)` call sites: unsigned packages now
   raise `UnsignedPackageRejected` by default; pass `accept_unsigned=True`
   per call to opt in (audit-event `PACKAGE_INSTALL_UNSIGNED` records
   the decision).

## SECURITY-BREAKING: E-34 (signing-kms)

- **Before**: `HmacDevSigning()` silently fell back to a hard-coded dev
  default if no `FLOWFORGE_SIGNING_SECRET` env var was present.
- **After**: same call raises
  `RuntimeError("explicit secret required …")`.

### Migration paths (pick one)

| Path | Action | When to use |
|---|---|---|
| **A — fix forward** | Set `FLOWFORGE_SIGNING_SECRET` from your secrets store. | Default. Do this. |
| **B — bridge for one minor** | Set `FLOWFORGE_ALLOW_INSECURE_DEFAULT=1` on the host. Loud `WARNING` log + `flowforge_signing_secret_default_used_total` Prometheus counter increments per process. Schedule the real fix within one minor. | Old prod images, customer-managed deploys. |
| **C — KMS rotation upgrade** | Migrate to `HmacDevSigning(keys={kid: secret}, current_key_id=kid)` or to a real KMS via `AwsKmsSigning` / `GcpKmsSigning`. | Anyone treating dev signing as production. |

### Pre-deploy verification

```bash
flowforge pre-upgrade-check signing
```

Exit codes:
- `0` + `PASS`: explicit secret set; safe to deploy.
- `0` + `WARN`: bridge flag set; deploy but plan the real fix.
- `1` + `FAIL`: no secret, no bridge; refuse to deploy.

CI ratchet `scripts/ci/ratchets/no_default_secret.sh` keeps the legacy
default from being reintroduced.

## Schema migration: audit-pg ordinal column (E-37)

Fresh deploys built from `flowforge_audit_pg.sink.create_tables()`
already have the new `ordinal BIGINT NULL` column +
`UNIQUE(tenant_id, ordinal)` constraint.

Existing PG-backed environments need a 3-step idempotent migration:

```bash
# 1. Add the column. Fast, app-safe (readers fall back to occurred_at
#    ordering when ordinal IS NULL).
python -m flowforge_audit_pg.migrations.audit_ordinal_backfill \
  --dsn "$AUDIT_DSN" --step add-column

# 2. Backfill ordinals per tenant under per-tenant advisory lock.
#    Online; safe to run during light load.
python -m flowforge_audit_pg.migrations.audit_ordinal_backfill \
  --dsn "$AUDIT_DSN" --step backfill --batch-size 10000

# 3. Add the unique constraint. Fast.
python -m flowforge_audit_pg.migrations.audit_ordinal_backfill \
  --dsn "$AUDIT_DSN" --step add-constraint

# 4. Verify. Should report 0 null ordinals + constraint present.
python -m flowforge_audit_pg.migrations.audit_ordinal_backfill \
  --dsn "$AUDIT_DSN" --step verify
```

Rollback: drop the unique constraint + column in reverse order. Audit
chain hash is unaffected (ordinal is metadata, not part of the
canonical-body bytes).

## Behavioural changes

### E-32 — engine fire is per-instance serialised

- Concurrent `fire()` for the same `instance.id` now raises
  `ConcurrentFireRejected`.
- Outbox / audit failure rolls back the in-memory `Instance` snapshot.
- New exceptions: `ConcurrentFireRejected`, `OutboxDispatchError`.

### E-35 — op registry frozen at module init

- Calling `register_op(...)` after import-time raises
  `RegistryFrozenError`.
- Each op declares an arity at registration; mismatched calls raise
  `ArityMismatchError`.
- Validators reject the bad arity at compile time, not runtime.

### E-36 — tenancy SQL hardening

- GUC keys must match `^[a-zA-Z_][a-zA-Z_0-9.]*$`.
- `bind_session()` asserts the session is in a transaction; raises
  outside of one.
- Elevation flag is now per-task (`ContextVar`), not module-global.

### E-37 — audit-chain hardening

- `record()` now serialises per-tenant via `pg_advisory_xact_lock`.
- New `UNIQUE(tenant_id, ordinal)` constraint catches duplicates at the
  DB layer.
- `verify_chain()` streams in chunks (default 10K rows). Memory peak
  bounded.

### E-37b — hub trust gate

- Default install path raises `UnsignedPackageRejected` for unsigned
  packages.
- Opt in per call: `install(..., accept_unsigned=True)`. Emits
  `PACKAGE_INSTALL_UNSIGNED` audit event.
- Error messages no longer leak `key_id` internal values.

### E-41 — fastapi + WS hardening

- CSRF cookie default is `secure=True`. Setting `secure=False` raises
  `flowforge_fastapi.ConfigError` unless paired with `dev_mode=True`.
- WS auth uses the new `WSPrincipalExtractor` protocol; HTTP-scope
  spoof code path is removed.
- `WorkflowEventsHub` is per-app via `app.state` and dependency
  override — two FastAPI apps in one process never share subscribers.
- Cookie payload includes `iat` + `exp` (24h default; configurable via
  `ttl_seconds=`). Expired cookies rejected with 401.

### E-46 — workspace registration two-step

- All 45 Python packages are now registered as workspace members.
- 30 domain `flowforge-jtbd-*` packages carry `package = false` so
  `uv build` discovers them but ships no wheel.
- The 15 strategic packages remain `package = true` and ship as before.

### E-58 — admin-token rotation (JH-04 split)

- `create_app(admin_token=...)` now accepts a comma-separated list.
  Each token is verified via `hmac.compare_digest`. Whitespace
  stripped. Single-token form remains backward-compatible.
- Full per-user RBAC tracked as E-73 (post-1.0). Design at
  `docs/design/E-73-jtbd-hub-rbac.md`.
