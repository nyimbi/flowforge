## §1 Verdict rationale

v1 absorbs the architect's 21 must-fix items with genuine substance. The V-1 BLOCKER (self-review masquerading as SME validation) is properly struck: Option B's "same Claude agent acting as 30 SMEs" language is gone from §1.3, Tier-A/Tier-B distinction is in the principles, and the signoff block at §15 carries machine-readable `reviewer_credentials: ai-author`, `tier: B`, and `human_sme_review: pending` fields. The 5 open questions from v0 §13 are answered, not posed. The critical-path math is corrected (6 wk, X1→X2→X3 sequential, no idle gap). The 8-scenario pre-mortem is present. All 8 invariants are required-green per X3 PR.

However, three concrete deficiencies survive architect-review absorption, two of which are MAJOR:

1. **Ticket ID collision (MAJOR):** `E-78b` is used for two distinct work items — the engineering snapshot ticket (§5.1, X2 sprint) and the second Tier-B content batch (v0.2.5, §4.2). An executor following this plan will have ambiguous target for every CI reference to `E-78b`. This is an unambiguous naming error that will corrupt signoff-checklist row lookup, `check_signoff.py` output, and PR tagging.

2. **Soak runner file referenced but does not exist (MAJOR):** `make v020-soak-forks` calls `uv run python framework/tests/soak/run_soak.py`, but `framework/tests/soak/` does not exist in the repository. Gate 7 (the X3 hard exit gate) depends on this script. Referencing a non-existent file as a hard exit gate means the gate is unrunnable as written — G-5 fails.

3. **Tier-A named SME signer is a `<name>` placeholder throughout (MINOR):** §5.2 Tier-A acceptance demands `signer: <name>` and §4.2 requires "named insurance SME + product lead sign" for v0.2.1. The plan acknowledges single-stakeholder governance (audit-2026 signoff-checklist roles header), but the specific human identity for each of the 5 Tier-A domain SMEs is not committed in the plan. The audit-2026 precedent named Nyimbi Odero in every SME role slot at signoff time. This is a MINOR because audit-2026 established that single-stakeholder governance is acceptable and documented, not a structural gap — but the plan should explicitly state which human fills each Tier-A SME slot at sprint kickoff rather than leaving `<name>` as a deferred placeholder.

No new BLOCKERs survive. All gates are evaluated below.

**Verdict: ITERATE** — one Partial-that-is-actually-a-naming-error (E-78b collision) and one non-existent soak script file must be resolved before this plan is executable. These are concrete, small fixes. v2 should be achievable in a single pass.
