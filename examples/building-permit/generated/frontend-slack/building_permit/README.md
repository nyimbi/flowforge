## What this is

A shell-only Slack adapter for the `building-permit` bundle. The
generator emits:

* `commands.py` — slash-command catalog (one slash per JTBD with a
  per-event subcommand grammar).
* `messages.py` — Block Kit interactive-message templates keyed by
  audit topic.
* `router.py` — request handler that translates a parsed slash command
  into a POST against `/<url-segment>/events`.

## What this is *not*

The generator emits the **routing skeleton only**. Hosts wire the
concrete Slack bot themselves:

* webhook URL and signing secret (request signature verification);
* event delivery (Socket Mode, Events API, or proxy of choice);
* user-to-tenant identity bridging.

That intentional cut keeps the bundle deterministic and lets each host
pick its own Slack runtime.

## Operation source

Every slash command is pinned to the same `openapi.yaml` the W1
generator emits at the bundle root (`../../openapi.yaml` relative
to this package). The adapter does **not** re-derive operations from
the bundle.

## Wiring

Hosts subclass `SlackAdapter` from `router.py` and implement two
hooks:

* `verify_signature(headers, body)` — Slack-Bolt-style signature
  check.
* `resolve_principal(slack_user_id, slack_team_id)` — bridge Slack
  identity to the bundle's tenant + user model.

Then call `dispatch(request)` from your bot's HTTP handler. The router
takes care of slash-command parsing and POSTing to the right backend
endpoint.
