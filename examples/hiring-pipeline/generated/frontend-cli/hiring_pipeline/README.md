## What this is

A scriptable Python CLI that fires workflow events into the bundle's
FastAPI backend. Built on [Typer](https://typer.tiangolo.com/) and an
async [httpx](https://www.python-httpx.org/) client.

## Why a CLI alongside the webapp

Enterprise ops, customer-success, and CI smoke-test surfaces all want a
non-browser handle on the workflow. Coupling them through the
customer-facing Next.js frontend would force operators onto the wrong
stack; a separate Python CLI keeps each surface deployable on its own
release cadence.

## Operation source

Every command is pinned to the same `openapi.yaml` the W1 generator
emits at the bundle root (`../../openapi.yaml` relative to this
package). The CLI does **not** re-derive operations from the bundle —
when the spec changes, regenerate this CLI in lockstep.

## Usage

The top-level entry point is the `hiring-pipeline-cli` console
script (see `[project.scripts]` in `pyproject.toml`):

```bash
hiring-pipeline-cli --help
```

Each JTBD becomes a Typer subapp; each transition event becomes a
command under that subapp:

### Complete hire and onboard (`complete-hire`)
* `hiring-pipeline-cli complete-hire approve`
* `hiring-pipeline-cli complete-hire reject`
* `hiring-pipeline-cli complete-hire submit`
### Conduct interview loop (`conduct-interview`)
* `hiring-pipeline-cli conduct-interview approve`
* `hiring-pipeline-cli conduct-interview submit`
### Extend an offer (`extend-offer`)
* `hiring-pipeline-cli extend-offer approve`
* `hiring-pipeline-cli extend-offer escalate`
* `hiring-pipeline-cli extend-offer reject`
* `hiring-pipeline-cli extend-offer submit`
### Screen a candidate (`screen-candidate`)
* `hiring-pipeline-cli screen-candidate approve`
* `hiring-pipeline-cli screen-candidate submit`
### Source a candidate (`source-candidate`)
* `hiring-pipeline-cli source-candidate approve`
* `hiring-pipeline-cli source-candidate reject`
* `hiring-pipeline-cli source-candidate submit`

The CLI POSTs to `/<url-segment>/events` exactly as the OpenAPI spec
documents. Every `data_capture` field becomes a `--<field>` Typer option;
required fields are required. Override the backend URL with
`--base-url` (default: `http://localhost:8000`).
