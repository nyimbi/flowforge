# flowforge-rbac-static

Static RBAC resolver backed by a YAML or JSON role-permission map.

Part of [flowforge](https://github.com/nyimbi/ums/tree/main/framework) — a portable workflow framework with audit-trail, multi-tenancy, and pluggable adapters.

## Install

```bash
uv pip install flowforge-rbac-static
```

## What it does

Reads a config dict (or YAML/JSON file) that maps roles to permission lists and principals to roles. `has_permission` checks walk the principal's roles and return `True` if any role grants the requested permission. System principals (`Principal.is_system=True`) bypass the check and always return `True`.

This resolver holds its entire permission map in process memory. It has no external dependencies at check time, which makes it fast and suitable for small deployments, CLI tools, demos, and the default project skeleton generated by the JTBD generator. It does not support dynamic grants, relationship-based access, or multi-tenant permission delegation — use `flowforge-rbac-spicedb` for those cases.

`assert_seed(names)` compares a list of expected permission names against the loaded catalogue and returns any that are missing. Pass `strict=True` to raise `CatalogDriftError` instead.

## Quick start

```python
from flowforge_rbac_static import StaticRbac
from flowforge.ports import Principal, Scope

# Load from a YAML file — path is allowlist-rooted (RB-01)
rbac = StaticRbac.from_yaml(
	"./rbac.yaml",
	allowed_root="./",   # rejects ../traversal paths
)

principal = Principal(user_id="alice", roles=())
scope = Scope(tenant_id="t-1")

allowed = await rbac.has_permission(principal, "claim.create", scope)
```

Config file shape:

```yaml
roles:
  intake_clerk: [claim.create, claim.read]
  triage_officer: [claim.read, claim.update]
principals:
  alice: [intake_clerk]
  bob: [triage_officer]
permissions:
  - {name: claim.create, description: Create a new claim}
  - {name: claim.read,   description: Read a claim}
  - {name: claim.update, description: Update a claim}
```

Wire it into flowforge config:

```python
from flowforge import config

config.rbac = StaticRbac.from_yaml("./rbac.yaml")
```

## Public API

- `StaticRbac(config, *, strict=False)` — construct from a plain dict
- `StaticRbac.from_yaml(path, *, strict=False, allowed_root=None)` — load from YAML
- `StaticRbac.from_json(path, *, strict=False, allowed_root=None)` — load from JSON
- `CatalogDriftError` — raised by `assert_seed` in strict mode when expected permissions are absent

## Configuration

No environment variables. All configuration is in the YAML/JSON file passed to the constructor.

## Audit-2026 hardening

- **RB-01** (E-55): `from_yaml` and `from_json` accept an `allowed_root` parameter; paths are resolved with `Path.resolve()` and then checked against the root via `relative_to()`; `../` traversal strings that escape the root raise `ValueError` before the file is opened

## Compatibility

- Python 3.11+
- `pyyaml` (only for `from_yaml`; not required if you use `from_json` or the dict constructor)
